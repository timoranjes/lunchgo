# DATA DIRECTORY

## OVERVIEW
Restaurant data for Hong Kong — district-chunked JSON files generated by Python pipeline (`scripts/enrich_data.py`).

## STRUCTURE
```
data/
├── district_index.json      # Index manifest — lists all district files + restaurant counts
├── district_*.json          # 18 district files (central_western, eastern, islands, etc.)
├── hk_restaurants.json      # Deprecated: legacy full dump
└── hk_restaurants_v2.json   # Deprecated: v2 format
```

## WHERE TO LOOK
| Task | File | Notes |
|------|------|-------|
| District index | `district_index.json` | Maps district IDs to chunk filenames + counts |
| Per-district data | `district_*.json` | Compact array format, not object-per-record |
| Full dataset | `hk_restaurants.json` | Deprecated; regenerated by pipeline |

## DATA SCHEMA (district_*.json)
Compact array format — each restaurant is an array, not an object:
```
[id, name, name_en, lat, lng, address, rating, user_ratings_total, price_level, types, cuisine, photos, place_id]
```
- `id`: unique identifier (`place_<place_id>` or random)
- `lat/lng`: floating point coordinates
- `rating`: 0-5 float; `user_ratings_total`: integer
- `price_level`: 0-4 integer
- `types`: array of Google Places types
- `cuisine`: comma-separated string
- `photos`: array of photo URLs (max 5)
- `place_id`: Google Places place_id

## CONVENTIONS
- **District naming** — snake_case, lowercase (`central_western`, `kowloon_city`)
- **File naming** — `district_<district_name>.json`
- **Index format** — `district_index.json` contains `{ "<district_id>": { "file": "...", "count": N } }`
- **No manual edits** — all data generated by pipeline; manual changes will be overwritten by daily CI

## ANTI-PATTERNS
- **Do NOT manually edit JSON** — pipeline overwrites daily via GitHub Actions
- **Do NOT use `hk_restaurants.json`** — deprecated; use district files or run pipeline
- **Do NOT change schema** — compact array format is intentional for size; app parser depends on position

## PIPELINE
- Source: FEHD (govt restaurant licenses XML) + Overpass API (OSM)
- Merge strategy: exact name match → proximity + name similarity
- Output: district-chunked JSON + `district_index.json`
- Run: `python scripts/enrich_data.py`
